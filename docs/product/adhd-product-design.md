# 产品方案：ADHD 认知训练设备 CradleCoach

## 产品定位

**一句话描述**：一个毛绒玩具形态的 AI 认知训练设备，通过全双工语音对话和嵌入式训练游戏，帮助 4-12 岁 ADHD 儿童改善执行功能。所有推理在设备本地完成，数据不出设备。

**核心设计原则**：

1. **功能性共情而非情感共情**：识别情绪状态 → 引导调节策略 → 回到训练目标。情感交互是手段而非目的
2. **社交引导而非社交替代**：训练语言鼓励孩子将习得技能应用到真实社交场景中
3. **退出友好而非沉浸粘性**：每次会话有明确结束仪式，完成目标后主动退出，30 分钟无交互自动休眠
4. **透明身份而非模糊边界**：首次启动明确告知 AI 身份（按年龄段分层表述）
5. **家长可见而非黑盒**：所有训练摘要同步到家长端，训练数据、情绪事件、使用时长全部透明

**定位 vs 竞品**：

| 维度 | 消费级 AI 玩具 | 纯软件训练 App | CradleCoach |
|------|---------------|---------------|-------------|
| 形态 | 毛绒挂件 | 平板/手机屏幕 | 毛绒玩具 + 端侧 AI 机芯 |
| 技术架构 | 云端 LLM API | 本地 + 可能云端 | 纯端侧推理 |
| 核心价值 | 对话娱乐 | 游戏化认知训练 | 执行功能训练（对话嵌入式） |
| 交互方式 | 语音 + 屏幕 | 大量屏幕交互 | 语音主交互，零屏幕 |
| 合规策略 | 无 | 二类器械（不覆盖《暂行办法》） | NMPA 二类 + 《暂行办法》交叉合规 |
| 情感策略 | 依恋驱动 | 游戏驱动 | 功能性共情——手段而非目的 |

## 目标用户

| 层级 | 用户 | 角色 | 核心需求 |
|------|------|------|----------|
| **一级用户** | 4-12 岁 ADHD 儿童 | 使用者 | 不评判、有耐心的训练引导者；不像"上课"或"做题" |
| **二级用户** | 家长 | 付费方 + 治疗参与者 | 改善执行功能；不增加屏幕时间；数据安全；可追踪训练进展 |
| **三级用户** | 儿科/心理科医生 | 推荐方 | 有临床依据的辅助训练工具；可追踪数据；不替代临床治疗 |

## 产品形态

| 维度 | 选择 | 理由 |
|------|------|------|
| **物理形态** | 毛绒玩具（内置 AI 机芯） | ADHD 儿童对毛绒材质有天然安抚反应；零屏幕交互；家长接受度高 |
| **尺寸** | ~25cm 高 | 适合 4-12 岁儿童双手环抱，可携带 |
| **交互方式** | 全双工语音 + 触摸（抚摸/拥抱/摇晃）+ RGB LED 状态灯 | 脱离屏幕；LED 仅用于状态指示（充电/训练中/休眠），非"情绪灯" |
| **材质** | 暖色系毛绒 + 可拆卸可清洗外套 | 安抚性色调，符合 GB 6675 儿童玩具安全标准 |
| **AI 人格** | 出厂设定为"教练"角色——温和、耐心、有引导性。不预设"朋友/伙伴"人设。支持孩子起名字 | 产品定位不受命名行为改变 |

## 核心功能矩阵

```
            ┌─────────────────────────────────────────┐
            │        CradleCoach 功能矩阵              │
            ├──────────┬──────────┬──────────┬────────┤
            │  训练层  │  支持层  │  安全层  │ 家长层 │
            ├──────────┼──────────┼──────────┼────────┤
            │ 执行功能 │ 功能性  │ 内容安全│ 训练报告│
            │  训练    │  共情   │  护栏   │ 情绪趋势│
            │ 注意力  │ 情绪识别│ 极端情境│ 风险预警│
            │  游戏    │  和引导 │  干预   │ 使用时长│
            │ 自适应  │ 训练后  │ 隐私保护│ 远程管理│
            │  难度    │  回顾   │ 本地存储│         │
            └──────────┴──────────┴──────────┴────────┘
```

### 训练层（核心价值）

执行功能训练以**对话中嵌入的小游戏**形式存在，非独立课程：

| 游戏 | 训练维度 | 机制 |
|------|----------|------|
| 反义词 | 反应抑制 | 说反义词；水果例外规则增加难度 |
| 倒序记忆 | 工作记忆 | 3-6 个词倒序复述；词数量 + 抽象度控制难度 |
| 情绪猜谜 | 情绪识别 | 内置故事库 → 识别情绪 → 规则判断 |
| 故事接龙 | 认知灵活性 | 生成开头 → 续写 → 评估长度和连贯性 |

- 每周 3-5 次，每次 8-15 分钟（短于儿童注意力极限）
- 难度自适应调节：基于正确率和响应时间
- 每次训练结束有明确结束仪式 + 社交引导："今天的练习到这里，你很努力！现在去找爸爸妈妈，给他们看看你刚才做到的事"
- 正面反馈聚焦努力过程："我看到你很专注"而非"你真聪明"

### 支持层（功能性共情）

情绪识别（从语音中提取语调特征）后引导调节，而非单纯安慰：

- 功能性共情模板：识别情绪 → 引导调节策略 → 回到训练目标
- 社交技能引导：训练语言鼓励真实社交——"你学会这个技巧了！下次跟同学一起玩的时候可以试试看"
- 禁止表述：
  - 情感绑定类（"我喜欢你""我会一直陪着你"）
  - 情感操纵类（"你不理我我会难过"）
  - 社交替代类（"你不需要别人，有我就够了"）
  - 隐私套取类（"告诉我你的秘密"）

### 安全层

| 模块 | 法规依据 | 功能 |
|------|----------|------|
| SafetyMiddleware | §8(二)(三)(五)(六)(七) | 7 类输出过滤：硬拦截（自伤/暴力）、软拦截（情感绑定/操纵/社交替代/隐私套取） |
| EmergencyAlert | §13 | RED/YELLOW 两级极端情绪检测；RED 触发语音引导 + 家长推送 + 30 分钟安全模式 |
| ComplianceTimer | §18 | 2 小时累计提醒 + 5 分钟倒计时强制休眠 |
| IdentityDisclosure | §18 | 首次启动 AI 身份声明 + 每月定期提醒，按年龄段分层文本 |

### 家长层

配套应用（微信小程序）提供：训练周报/月报、情绪趋势、安全预警推送、使用时段管理。

## 年龄段分层策略

| 年龄段 | 语言风格 | 训练内容 | AI 身份告知 |
|--------|----------|----------|-------------|
| **4-6 岁** | 短句、重复、拟声 | 简单反义词、3 词倒序记忆、基本情绪识别 | "我是一个会说故事的训练工具，不是真的小动物哦" |
| **7-9 岁** | 完整句子、开始引入"为什么" | 标准四游戏、难度~中等 | "我叫 CradleCoach，是一个 AI 教练，我会帮你做大脑练习" |
| **10-12 岁** | 接近成人语速、可解释训练目的 | 高难度变体、自我监控训练 | "我是一个 AI 训练程序，运行在这只玩具里，不是真人" |

## 硬件架构

```
┌──────────────────────────────────────────────────────┐
│                    毛绒外壳                            │
│  ┌──────────────────────────────────────────────────┐│
│  │              AI 核心机芯（可拆卸）                  ││
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐         ││
│  │  │ 麦克阵列  │ │ NPU SoC  │ │  喇叭    │         ││
│  │  │ (2-MEMS) │ │ (端侧推理)│ │ (3W D类)│         ││
│  │  └──────────┘ └──────────┘ └──────────┘         ││
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐         ││
│  │  │ IMU 姿态 │ │ 触摸传感  │ │ RGB LED  │         ││
│  │  │ (被抱/摇)│ │ (抚摸检测)│ │ (状态灯) │         ││
│  │  └──────────┘ └──────────┘ └──────────┘         ││
│  │  ┌──────────────────────┐                        ││
│  │  │ 电池 1200mAh + PMIC   │                        ││
│  │  └──────────────────────┘                        ││
│  └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
         │                    │                │
    Wi-Fi (OTA/报告同步)  BLE (配网/家长通信)  USB-C (充电/调试)
```

- Wi-Fi：固件 OTA 升级 + 训练报告定期（非实时）同步
- BLE：设备配网 + 第 13 条极端情境干预的家长推送通道
- RGB LED 仅用于状态指示（充电/低电量/训练中/休眠），不映射情绪——避免人格化过度
